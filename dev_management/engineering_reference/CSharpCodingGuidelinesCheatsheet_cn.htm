Coding Guidelines for C# 3.0, 4.0 and 5.0 Cheat Sheet
Design & Maintainability
(level 1 and 2 only)

基本原则

  • The Principle of Least Surprise最小惊奇原则
  • Keep It Simple Stupid保持简单
  • You Ain’t Gonne Need It你是不是以后才需要它
  • Don’t Repeat Yourself减少重复

Class Design(类设计)

  • 一个类或接口应该只有一个目的 (AV1000)
  • 一个接口应该小和集中 (AV1003)
  • 使用接口将类彼此解耦 (AV1005)
  • 不要用new关键字隐藏继承成员 (AV1010)
  • 应该将派生对象视为基类对象 (AV1011)
  • 不要从基类引用派生类 (AV1013)
  • 避免暴露对象依赖的对象(AV1014)
  • 避免双向依赖 (AV1020)
  • 类应该有状态和行为 (AV1025)


Member Design(成员设计)

  • 允许按任意顺序设置属性 (AV1100)
  • 不要使用互斥属性 (AV1110)
  • 方法或属性应该只做一件事 (AV1115)
  • 不要通过静态成员公开状态对象 (AV1125)
  • 返回一个IEnumerable<T> 或 ICollection<T> 代替具体的集合类 (AV1130)
  • 表示字符串或集合的属性、方法和参数不应该是null (AV1135)
  • 尽可能具体地定义参数 (AV1137)

Miscellaneous Design(杂项设计)

  • 抛出异常而不是返回状态值 (AV1200)
  • 提供一个丰富而有意义的异常消息文本 (AV1202)
  • 不要通过捕获一般异常来掩盖错误 (AV1210)
  • 总是检查事件处理程序委托是否为null (AV1220)
  • 正确处理异步代码中的异常 (AV1215)
  • 使用受保护的虚拟方法来引发每个事件 (AV1225)
  • 在引发事件时，不要将null作为发送方参数传递。 (AV1235)
  • 如果适用，使用泛型约束 (AV1240)
  • 在返回LINQ结果前应先对该LINQ表达式求值 (AV1250)


Maintainability(可维护性)

  • 方法不应超过7个语句 (AV1500)
  • 使所有成员都为private，并默认为internal类型 (AV1501)
  • 避免双重否定的条件 (AV1502)
  • 不要用 "magic numbers(魔法数字)" (AV1515)
  • 只有当类型非常明显时才使用var (AV1520)
  • 尽可能晚地声明和初始化变量 (AV1521)
  • 在使用Object和Collection使用初始化器时尽量写在单独的语句中 (AV1523)
  • 不要对 true 或 false进行显式的比较 (AV1525)
  • 不要在 for 或 foreach 循环中改变循环变量 (AV1530)
  • 避免嵌套循环 (AV1532)
  • 在 if, else, while, for, foreach and case后总是添加一个块 (AV1535)
  • 在switch的最后一个case后，总是添加一个default块 (AV1536)
  • 每个 if-else-if声明后都应该用else-部分，来使整个语句更完整 (AV1537)
  • 要厌恶使用多个return语句 (AV1540)
  • 不要用 if-else 语句代替简单的（条件）赋值。 (AV1545)
  • 在方法或属性中封装复杂表达式 (AV1547)
  • Call the most overloaded method from other overloads (AV1551)
  • 只使用可选参数替换重载。 (AV1553)
  • 避免使用命名参数 (AV1555)
  • 不允许超过三个参数的方法和构造函数 (AV1561)
  • 不要用 ref 或 out 参数 (AV1562)
  • 避免方法带一个 bool 标志 (AV1564)
  • 对于带as的运算一定要检查其结果 (AV1570)
  • 不要注释代码(用版本管理工具) (AV1575)


Framework Guidelines(框架指南)

  • 用C #类型别名代替 System 命名空间的类型 (AV2201)
  • 以最高警告级别构建 (AV2210)
  • 使用lambda表达式代替委托 (AV2221)
  • 仅在与动态对象交互时使用dynamic关键字 (AV2230)
  • 相对于 Task 优先使用async/await (AV2235)

Naming(命名)

  • Use US English (AV1701)
  • 不要在变量、参数和类型成员中使用数字 (AV1704)
  • 不要给字段加前缀 (AV1705)
  • 不要使用缩写 (AV1706)
  • 命名成员、参数或变量应根据其含义而不是其类型 (AV1707)
  • 命名类型应使用名词、名词短语或形容词短语 (AV1708)
  • 不要在类或枚举的成员中重复其(类或枚举)名称 (AV1710)
  • 避免与其他名字相混淆的短名称或名称 (AV1712)
  • 方法的命名使用“动词-对象”组合 (AV1720)
  • 对命名空间命名时，使用名称、层、动词和功能 (AV1725)
  • 用下划线表示不相关的lambda参数 (AV1739)

Documentation(文档、注释)

  • Write comments and documentation in US English (AV2301)
  • 对所有的public, protected 和 internal 类型和成员写注释 (AV2305)
  • 避免(方法)内嵌注释 (AV2310)
  • 只对复杂的算法或决策写注释 (AV2316)
  • 不要使用注释来跟踪稍后要做的工作。 (AV2318)

Layout(布局)

  • 最大行长度为130个字符。
  • 缩进4个空格, 不要用 Tabs
  • 在关键词及其表达式之间保持一个空格，例如 if , 但是不要在 (后和 )前加空格。
  • 在运算符两边加空格，例如 +, -, ==, 和其他。
  • 总是在关键字if, else, do, while, for and foreach之后添加圆括号。
  • 总是把开始和结束括号放在新行上。
  • 不要缩进对象初始器，并且每行初始化一个属性。
  • 不要缩进lambda表达式
  • 把整个LINQ语句在同一行，或者每个关键字在同一缩进位置。
  • 在比较条件两边添加括号，但不要在单个条件时添加括号。

Empty lines(空行)

  • 成员之间
  • 在结束括号之后
  • 多行语句之间
  • 不相关代码块之间
  • 在 #region 两侧
  • 在不同根命名空间的using语句之间

Member order(成员顺序)

  1. 私有字段和常量
  2. 公共常量
  3. 公共只读静态字段
  4. 工厂方法
  5. 构造函数和析构函数
  6. 事件
  7. 公共属性
  8. 调用顺序中的其他方法和私有属性

Important Note

These coding guidelines are an extension to Visual Studio's Code Analysis functionalty, so make sure you enable that for all your projects. Check the full document for more details.
